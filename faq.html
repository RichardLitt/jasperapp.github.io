Architecture

The architecture of Jasper is made two parts.

  • Polling issues in background: Fetch issues by search API of GitHub. And store up issues to SQLite.
  • Display issues in frontend: Take issues from SQLite. And display issues using Electron.

Merged Pull Request

Jasper can not distinguish a merged pull request from a closed pull request. Because the search API of GitHub responses only "open" and "closed", does not responses "merged". Sorry...

Lost Issues

Jasper can not fetch all past issues of you. Jasper will fetch recently issues of several hundred. For this reason, in some case Jasper may lose a issue.
